# Contributing

We welcome contributions! All development work is organized through GitHub Issues, which track bugs, features, and tasks.

## Getting Started

1. Find an issue to work on from the [GitHub Project Board](https://github.com/orgs/dggpoliticalaction/projects/3)
2. Create a branch off `main` with a descriptive name (e.g., `123-fix-landing-page`, `setup-notification-system`)
3. Make your changes and commit them with clear, concise messages (e.g., "Fixes landing page navigation", "Sets up notification collection")
4. Push your branch and create a pull request (PR)
5. Address review feedback and iterate on your PR
6. Once approved, your PR will be merged into `main`

## Tips

- **Branch naming**: Use descriptive names that indicate the purpose of the branch
- **Commit messages**: Write short, clear sentences describing what the commit does
- **Auto-close issues**: Include `Closes #123` in your PR description to automatically close the related issue when merged
